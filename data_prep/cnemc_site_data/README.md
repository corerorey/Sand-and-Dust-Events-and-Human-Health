### 原始文件格式 (Raw File Format)

每日文件命名规则：`china_sites_YYYYMMDD.csv`（UTF-8 编码）

文件为**长表格式（long format）**，列结构如下：

```
date,   hour, type,   1001A, 1002A, 1003A, ...
20140513,  1, PM2.5,    93,    45,    78, ...
20140513,  1, PM10,    103,    61,    95, ...
20140513,  1, AQI,     123,    72,   110, ...
...
```

- `date`：8 位整数日期，如 `20140513`
- `hour`：小时（1–24，**注意：原始数据小时从 1 起计**）  
  *(Hour field uses 1-based indexing in raw files)*
- `type`：污染物类型字段，取值见上表
- 后续各列：以**监测点编码（site code）**为列名，值为对应污染物浓度

### 注意事项 (Caveats)

- 某些日期因服务器故障或数据来源问题可能**完全缺失**，不会生成当日文件  
  *(Some dates may be entirely missing due to server issues or upstream data gaps)*
- 同一监测点在某年若缺失率 ≥ 95%，本项目 `prepro.py` 中会将该年数据整体置为 NaN  
- 短缺口（≤ 2 小时连续缺失）通过时间线性插值填补；更长的缺口保留为 NaN（科学保守策略）  
  *(Short gaps ≤ 2 h are filled by time-based linear interpolation; longer gaps are kept as NaN)*
- 原始 CSV 使用 UTF-8 编码；用 Excel 直接打开时需通过"数据 → 文本导入"并指定 UTF-8，否则中文显示乱码
